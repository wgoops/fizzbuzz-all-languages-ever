                Paul, if your still maintaining this web site, I had some questions for you
                regarding how you got the Xterm keymapping to work properly under linux. 

                I was able to install the APL fonts ok and they show up fine using the linux
                command:

                xterm -fn kaplgallant-19

                (Then under the xterm window, I type in A+ to get into the A+ interpreter.)

                The problem is that when I type an APL character in under A+,
                it does not show up on the screen because of the Xterm mapping.

                The A+ install, installed an Xterm file under the /usr/local/app-defaults
                directory which contains mappings for the keyboard to the APL
                character needed to be produced on the screen. 

                The Linux command I used was

                xrdb -load XTerm

                to load up the A+ APL XTerm keymapping file.

                This did not seem to produce anything in terms of mapping
                the APL characters successfully.  

                Note that I can still type expressions under the A+, without
                the APL characters showing up, and the expression will execute fine,
                however all the APL characters do not show up on the xterm screen. 

                Any help you could give me would be greatly appreciated. 

                If you see this post please e-mail me back at t.bracken@shaw.ca.

                Thanks for your help!

                Tom

                A further note to this. I was able to figure out what the problem was.
                It has to do with the encoding, which needs to be latin1 instead of UTF-8

                The linux script I have, which calls a+ needed to be changed to the following
                to export the correct encoding using the LANG environment variable.

                #
                # The LANG variable has to be changed from en_US.UTF-8 to en_US.latin1
                #
                # This allows the APL characters to be displayed properly.
                #
                # T. Bracken Jan 23,2012
                #
                export LANG=en_US.latin1
                xterm -fg cyan -bg black -fn kaplgallant-19 -e a+

                Now the APL character set shows up just fine.

                Another point is that the A+ source that was given at the A+
                website has problems compiling, and it took me 1 to 2 hours
                to locate the patches that were applied by the developer but
                where never included in the source!!
                Below are the fixes which need to be applied to two C programs
                src/MSTypes/MSFloat.C and
                src/MSTypes/MSUtil.C
                (A plus on the side means add this line, and a minus on the side means delete
                the line)

                MSFloat.C

                - char *np,buf[512];
                + char buf[512];
                + const char *np;

                MSUtil.C
                int places,len,i,j;
                - char *cp;
                + const char *cp;

                    # 5. Run a custom command instead of my shell: enabled
                    # 6. Custom command: luit -encoding 'ISO 8859-1' -- /usr/bin/a+
